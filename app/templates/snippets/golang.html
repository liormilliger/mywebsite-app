Go (Golang): Concurrency-Driven Booking Application

As I expand my DevOps toolkit, I've integrated Go for building high-performance CLI tools and microservices. This snippet demonstrates a booking application that leverages Go's powerful concurrency model.

  • Strong Typing & Structs: Uses custom UserData types to manage complex data structures cleanly.
  • Concurrency: Implements go routines and sync.WaitGroup to simulate asynchronous tasks, such as sending emails, without blocking the main execution thread.
  • Input Validation: Features robust logic for validating user strings, email formats, and ticket availability before processing transactions.

package main

import (
    "fmt"
    "sync"
    "time"
)

type UserData struct {
    firstName       string
    lastName        string
    email           string
    numberOfTickets uint
}

var wg = sync.WaitGroup{}

func main() {
    // ... (Greeting and Input logic)
    if isValidName && isValidEmail && isValidTicketNumber {
        bookTicket(userTickets, firstName, lastName, email)

        // Start a goroutine for asynchronous processing
        wg.Add(1)
        go sendTicket(userTickets, firstName, lastName, email)

        wg.Wait()
    }
}

func sendTicket(userTickets uint, firstName string, lastName string, email string) {
    time.Sleep(10 * time.Second)
    var ticket = fmt.Sprintf("%v tickets for %v %v", userTickets, firstName, lastName)
    fmt.Printf("Sending ticket: %v to email address %v\n", ticket, email)
    wg.Done()
}
