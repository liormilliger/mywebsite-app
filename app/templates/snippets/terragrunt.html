Terragrunt: Infrastructure Orchestration & DRY Principles

This project represents a comprehensive self-study course I developed to master Terragrunt. It demonstrates how to wrap Terraform to create maintainable, production-grade infrastructure by following the "Don't Repeat Yourself" (DRY) philosophy.

Core Project Goal: Deploying a full production stack (VPC + EKS) using modular inheritance and dependency management.
GitHub Repository: github.com/liormilliger/terragrunt-course.git

Course Progression & Key Milestones

  • Stage 1: The DRY Architecture - Implementing Parent-Child inheritance to manage multiple environments (Dev/Stage/Prod) without duplicating provider or backend blocks.
  • Stage 2: Remote State Management - Automating the creation of S3 backends and DynamoDB locks, ensuring unique state keys using path_relative_to_include().
  • Stage 3: Dependencies & Stack Connection - Using dependency blocks to pass outputs between isolated modules (e.g., passing VPC IDs to an EKS module).
  • Stage 4: Production Capstone - Deploying a complete AWS VPC and EKS cluster using official Registry modules via tfr:/// and orchestrating the stack with terragrunt run-all.

Project Structure Snippet

# Root terragrunt.hcl (Parent)
remote_state {
  backend = "s3"
  config = {
    bucket         = "my-terraform-state-${get_aws_account_id()}"
    key            = "${path_relative_to_include()}/terraform.tfstate"
    region         = "us-east-2"
    encrypt        = true
    dynamodb_table = "terraform-locks"
  }
}

# Child module (live/dev/eks/terragrunt.hcl)
include "root" {
  path = find_in_parent_folders()
}

dependency "vpc" {
  config_path = "../vpc"
}

inputs = {
  vpc_id     = dependency.vpc.outputs.vpc_id
  subnet_ids = dependency.vpc.outputs.private_subnets
}
