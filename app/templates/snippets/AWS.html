My experience with AWS grows with time, and thanks to AWS giving credits for development, I can expand my knowledge and be an AWS agent.

AWS Services used for this website:

  • VPC - Isolated cloud network for all resources.
  • EC2 - Virtual servers (nodes) for the EKS cluster.
  • EKS - Managed Kubernetes for hosting the application.
  • IAM - Managed permissions using least-privileged policies.
  • Security Groups - Instance-level firewall for minimal access.
  • S3 - Storing the terraform.tfstate file securely.
  • Route 53 - DNS resolution and routing for the website.

Other services I am experienced with:

  • Lambda - Serverless functions for backend automation.
  • SQS - Message queue for decoupling services.
  • SNS - Pub/sub messaging and notification service.
  • DynamoDB - Managed NoSQL key-value database.
  • RDS - Managed relational databases (e.g., PostgreSQL).
  • Elastic Beanstalk - PaaS for quick deployment of web apps.
  • CloudFormation - Infrastructure as Code for AWS-native stacks.
  • CloudWatch - Monitoring, logging, and alerts for all services.
